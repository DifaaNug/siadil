Algoritma Penting

üî• Auto-Generate Document Number

üìÅ DocumentController.php (Line 68-116)

Format:

KODE_KATEGORI/TAHUN/BULAN/URUTAN
Contoh: SM/2026/02/0001

Algoritma Step-by-Step:

  1. Ambil kategori yang dipilih user ‚Üí Category::find($categoryId)
  2. Generate kode kategori (mapping hardcoded)
  3. Ambil tahun dan bulan saat ini ‚Üí date('Y'), date('m')
  4. Buat prefix ‚Üí SM/2026/02/
  5. Cari dokumen terakhir dengan prefix yang sama
  6. Ambil 4 digit terakhir ‚Üí substr($lastDoc->document_number, -4)
  7. Increment nomor ‚Üí $lastNumber + 1
  8. Padding 4 digit ‚Üí str_pad($number, 4, '0', STR_PAD_LEFT)

Code:

private function generateDocumentNumber($categoryId)
{
    $category = Category::find($categoryId);
    $categoryCode = $this->generateCategoryCode($category->name);

    $year = date('Y');   // 2026
    $month = date('m');  // 02
    $prefix = "{$categoryCode}/{$year}/{$month}/";

    // Cari dokumen terakhir
    $lastDoc = Document::where('document_number', 'like', $prefix . '%')
        ->orderBy('document_number', 'desc')
        ->first();

    if ($lastDoc) {
        $lastNumber = intval(substr($lastDoc->document_number, -4));
        $newNumber = str_pad($lastNumber + 1, 4, '0', STR_PAD_LEFT);
    } else {
        $newNumber = '0001';
    }

    return $prefix . $newNumber;
}

File Upload System

üìÅ DocumentController.php (Line 147-160)

Step-by-Step Upload:

  1. Validation - Tipe file & ukuran
  2. Generate Filename - Unique dengan UUID
  3. Store File - Ke storage/app/public/documents
  4. Save Metadata - Ke database

1. Validation

'file' => 'required|file|mimes:pdf,doc,docx,xls,xlsx,jpg,jpeg,png|max:10240'
// max:10240 = 10MB

2. Store File

if ($request->hasFile('file')) {
    $file = $request->file('file');

    // Generate unique filename
    $filename = Str::uuid() . '.' . $file->getClientOriginalExtension();
    // Contoh: a3f5c8d2-1234-5678-9abc-def012345678.pdf

    // Store ke storage/app/public/documents/
    $path = $file->storeAs('documents', $filename, 'public');

    // Save metadata ke database
    $validated['file_name'] = $file->getClientOriginalName();
    $validated['file_path'] = $path;
    $validated['file_type'] = $file->getMimeType();
    $validated['file_size'] = $file->getSize();
}

3. Create Storage Link

php artisan storage:link
// Creates: public/storage ‚Üí storage/app/public

4. Access File (Line 100-103)

üìÅ app/Models/Document.php
public function getFileUrlAttribute()

    return asset('storage/' . $this->file_path);

Soft Delete

üìÅ app/Models/Category.php (Line 14-15)

Konsep: Data tidak dihapus permanent, hanya ditandai "deleted"

use SoftDeletes;  // Import trait

// Soft delete
$category->delete();  // Set deleted_at = now()

// Query (auto exclude soft deleted)
Category::all();

// Include soft deleted
Category::withTrashed()->get();

// Restore
$category->restore();  // Set deleted_at = NULL

Q&A Penting

Q: Kenapa pilih Laravel?

A:

  • MVC Pattern (terstruktur)
  • Eloquent ORM (query mudah)
  • Security built-in (CSRF, SQL injection prevention)
  • Ecosystem besar (banyak package)
  • Development cepat (artisan commands)

Q: Jelaskan MVC Pattern

A:

  • Model: Data & business logic (Document.php)
  • View: Tampilan/UI (index.blade.php)
  • Controller: Bridge antara Model & View (DocumentController.php)

Flow: Route ‚Üí Controller ‚Üí Model ‚Üí View ‚Üí Response

A:

  • Hanya tambah timestamp di kolom deleted_at
  • Bisa di-restore kapan saja
  • Keuntungan: Recovery data, audit trail
